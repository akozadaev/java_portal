# legacy-module

Эта папка содержит намеренно плохой legacy-код для демонстрации рефакторинга.

Код не подключен к runtime основного Spring Boot-приложения.

## Что здесь намеренно сделано плохо

- Один класс `OldStatusUpdater` отвечает сразу за бизнес-правила, SQL, audit и email-текст.
- Статусы передаются строками.
- В коде много вложенных `if-else`.
- SQL собирается конкатенацией строк: `"SELECT * FROM docs WHERE id=" + id`.
- Статус меняется в нескольких местах одного метода.

## Зачем нужен characterization test

`OldStatusUpdaterCharacterizationTest` фиксирует текущее поведение legacy-кода перед рефакторингом.

Такие тесты нужны не для того, чтобы назвать старый код хорошим, а чтобы безопасно переписать его и проверить, что ключевые бизнес-сценарии не сломались.
