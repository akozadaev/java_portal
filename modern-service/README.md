# modern-service

Эта папка показывает отрефакторенную версию legacy-обработчика статусов.

Главная идея - вынести поведение, зависящее от статуса, из одного процедурного класса в небольшие strategy-классы. Сервис выбирает стратегию по текущему статусу и делегирует ей правила перехода.

## Что исправлено относительно legacy-module

- Строковые статусы заменены enum `DocumentStatus`.
- Вложенная procedural-логика разделена на strategy-классы.
- SQL-конкатенация удалена из бизнес-логики.
- Audit и notification возвращаются как результат бизнес-операции, а не смешиваются с SQL.
- Добавлена проверка пустого `documentId`.
- Добавлена явная обработка статуса `VALIDATING`.
- Добавлен facade `LegacyStatusUpdateService`, который принимает legacy-входные данные в старом формате.

## Стратегии

- `NewStatusStrategy` - правила для статуса `NEW`.
- `SentStatusStrategy` - правила для статуса `SENT`.
- `ValidatingStatusStrategy` - совместимое поведение для `VALIDATING`.
- `TerminalStatusStrategy` - поведение для завершенного документа.
- `UnknownStatusStrategy` - fallback для неподдержанных состояний.

## Совместимость с legacy

Для проверки совместимости `LegacyCompatibilityTest`.

Он сравнивает старую и новую реализацию по ключевым сценариям:

- успешная обработка `NEW`;
- невалидные документы;
- недоступный SMEV;
- отклонение оператором;
- уже завершенный документ;
- `VALIDATING`;
- неизвестный статус;
- старый строковый статус `ERROR`;
- пустой id документа.

Сравниваются бизнес-поля:

- итоговый статус;
- audit;
- notification/email.

SQL не сравнивается намеренно: в modern-реализации SQL-конкатенация удалена как архитектурная ошибка.

## Как запускаются тесты

`modern-service` не является отдельным runtime-модулем приложения.

Его исходники и тесты подключаются к Maven как test-source roots основного проекта через `build-helper-maven-plugin`.

Проверка:

```bash
make test-fast
```
